	++ зарегистрированный unisign: протокол (registry)

	Установку и настройку всего вышеперечисленного может взять на себя установщик данной утилиты (кроме ввода серийника CryptoPRO .NET)

	================================================================================
	
0) Клик по ссылке вида: <a href="unisign:https://test.ru?id=12345-45-54545-12">Sign this document</a>.
1) Запрос по указанному в ссылке URI для получения информации по текущей сессии.
2) Информация по текущей сессии приходит в виде XML (SigningInfo.xml).
3) На основе управляющей информации выбирается режим подписания (Smev2 | Smev3) (Enveloped | Side-by-side) (ID | No ID).
4) Исходный XML отображается пользователю в соответствующем поле.
5) Документ для подписания преобразовывается в человекочитаемый вид при помощи XSL Transformation и отображается в соответствующем поле.
6) Пользователь выбирает подпись из найденных на локальной машине или в хранилище текущего пользователя.
7) Пользователь нажимает большую кнопку ПОДПИСАТЬ!, вводит пассворд к контейнеру закрытого ключа для сертификата.
8) Подписанный документ отправляется на по тому же урлу, откуда пришла управляющая информация, но, например, с параметром signed=1.
9) На сервере подпись клиента проверяется (вопрос КриптоПРОшникам я задал) и файл сохраняется.
